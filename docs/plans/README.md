# Plans

This folder holds **implementation and architecture plans** for Memora (decisions, scope, file references, verification steps).

- Add new plans as dated or named Markdown files (e.g. `feature-name.md`).
- Keep plans **updated** when scope changes, or add a short “Superseded by …” note at the top of older files.

Why keep plans in the repo:

- **Team**: shared context without repeating background in every PR or chat.
- **Future work**: anyone (including AI assistants when you `@`-mention these files) can align with prior decisions and avoid redoing the same analysis.
